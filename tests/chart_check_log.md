## 1. 確認目的

`MTA v2.1` をベースに作成した `mta_touch_trigger_v1.pine` が、TradingView実機上で問題なく動作するかを確認した。

v1.0では、以下の流れが想定通り動くかを確認する。

```text
15分足MTAを検出
↓
1分足チャート上に15分MTAを表示
↓
価格が15分MTAに接触
↓
SETUP L / SETUP S を表示
↓
接触足の高値/安値を保存
↓
1分足終値で接触足高値/安値をブレイク
↓
ENTRY L / ENTRY S を表示
```

## 4. 実機確認前の注意

TradingViewでは、Pineエディタで保存しただけでは、チャート上の古いコンパイル済みスタディが残ることがある。

そのため、確認時は以下の手順を実施した。

```text
1. Pineエディタに修正版コードを貼り付け
2. 保存
3. チャート上の古いスタディを削除
4. マイスクリプトから再追加
5. 実機チャートで挙動確認
```

## 12. PINE2026-05 ルール確認

### 確認済み項目

- セミコロンによる複数文記述なし
- `//@version=5` 維持
- `DevLucem/ZigLib/1` をv5スクリプトで使用
- `request.security()` は `lookahead_off` / `gaps_off`
- UDTの値は `zpx / zix / ztm` に退避してから履歴参照
- `zP / zH / zB / zT` の4配列を最小サイズ計算に含めた
- `line.delete()` 後に `:= na`
- `label.delete()` 後に `:= na`
- 水平線は `x1 == x2` にしていない
- `for ... by -1` 不使用
