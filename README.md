# MTA Touch Trigger Lab

15分足MTAへの接触をセットアップとし、1分足トリガーでエントリー候補を表示する Pine Script インジケーター開発リポジトリ。

---

## 1. 開発目的

このリポジトリでは、既存の `MTA v2.1` をベースに、以下の流れで手法開発を行う。

```text
15分足MTAを検出
↓
1分足チャート上で15分MTAへの接触を判定
↓
接触後、1分足のトリガー条件を監視
↓
トリガー成立でエントリー候補を表示
```

まずはバックテスト用ストラテジーではなく、目視確認用のインジケーター版から開発する。

---

## 2. ベースコード

ベースコードは `MTA v2.1`。

主な特徴：

- `DevLucem/ZigLib/1` を使用
- `//@version=5`
- ZigZagピボットからMTAを算出
- チャート足MTAと選択足MTAを表示可能
- `request.security()` により上位足MTAを下位足チャートへ反映
- 上昇MTA / 下降MTA / 終了済みMTAを色分け表示
- MTA発生、MTA終了のアラート機能あり

---

## 3. 開発方針

### v1の目的

インジケーター版v1では、以下を実装する。

```text
15分足MTAに価格が接触したらセットアップ
↓
接触足の高値/安値を保存
↓
一定本数以内に1分足でブレイクしたらトリガー成立
↓
チャート上にSETUP / ENTRYラベルを表示
```

### 想定チャート

```text
チャート足：1分足
選択足TF：15分
```

### ロング想定

```text
15分MTAが上昇状態
かつ
1分足が15分MTAに接触
↓
接触足の高値を終値で上抜け
↓
ロングエントリー候補
```

### ショート想定

```text
15分MTAが下降状態
かつ
1分足が15分MTAに接触
↓
接触足の安値を終値で下抜け
↓
ショートエントリー候補
```

## 4. 初期v1仕様

### セットアップ条件

#### Long Setup

```text
sTr == 1
かつ
low <= sM
かつ
high >= sM
```

#### Short Setup

```text
sTr == -1
かつ
low <= sM
かつ
high >= sM
```

---

### トリガー条件

#### Long Trigger

```text
Long Setup発生後、
接触足の高値を終値で上抜け
```

#### Short Trigger

```text
Short Setup発生後、
接触足の安値を終値で下抜け
```

---

### 有効期限

初期値：

```text
60本
```

1分足チャートの場合、60分相当。

---

### 表示

v1では以下を表示する。

- 15分MTAライン
- SETUPラベル
- ENTRYラベル
- セットアップ足の高値/安値ライン
- トリガー待ち状態
- 有効期限切れ状態

## 5. 開発ルール

このリポジトリでは、PINE2026-05 のハマり集を前提にする。

特に以下を必ず守る。

### 構文ルール

- セミコロンで複数文を書かない
- 1行1文
- 関数内から外スコープ変数を書き換えない
- `line.delete()` / `label.delete()` 後は必ず `:= na`
- 水平線は `x1 == x2` にしない
- `for ... by -1` は使わない
- `for i = a to b` は `a > b` で逆走するため、必ず `start <= end` を確認する
- 空配列ループは禁止

### 配列ルール

- `.get()` / `.last()` 前にサイズ確認する
- 複数配列を並走させる場合は `math.min()` で最小サイズを境界にする
- `var int idx` などの保持indexは、アクセス前に必ず境界確認する

### 条件式ルール

- `or` / `and` の短絡評価に頼らない
- 配列アクセス、mapアクセス、na前提の計算は、必ずネスト `if` でガードする

### request.security ルール

`request.security()` は必ず以下を指定する。

```pine
lookahead = barmerge.lookahead_off
gaps = barmerge.gaps_off
```

### ライブラリルール

- `DevLucem/ZigLib/1` を使う場合は、原則 `//@version=5` を維持する
- ライブラリUDTの履歴参照は慎重に扱う
- 必要値は float / int 系列に退避してから `[1]` 参照する

## 6. TradingView確認ルール

`pine_check` / `pine_compile` が通っても完了とはみなさない。

必ずTradingView実機で確認する。

特に以下を確認する。

- ランタイムエラーが出ないか
- 古いスタディが残っていないか
- 修正後はチャート上のスタディを削除してから再追加する
- MTAラインが想定通り表示されるか
- SETUPラベルが接触位置で出るか
- ENTRYラベルがトリガー位置で出るか
- アラート条件が想定通りか
